que es git
es chiquito y poderoso
es distribuido DVCS
tiene staging area
es abierto y todos puedes participar
es el mas usado
PERFORMANCE
esta optimizado para ejecutar diferentes acciones
no se confunde por el nombre de los archivos porque se enfoca en el contenido
SEGURIDAD
su prioridad es manejar el codigo fuente
aseugrado criptograficamente
FLEXIBILIDAD
soporta desarrollo no lineal
util en proyectos pequeños o grandes
compatibilidad con  sistemas existentes y protocolos
full seguimiento de ramas y etiquetas asi como de merging o reverting

por qué git
el uso de ramas permite que el main SOLO contenga software de calidad
compatible con agilidad porque permite conectar con jira con una rama
cada developer tiene acceso a todo pues tiene su propio repositorio lo que permite trabajar siempre en lo suyo aunque alguien mas haya cometido un error
pull request es solicitar a otro desarrollador hacer merge de tu rama dentro de su repositorio lo que permite conversaciones entre los desarrolladores antes de integrar su trabajo al codigo base
ventaja que casi todos ya saben usar git y sistemas distribuidos
facilita el workflow agil y permite que sea rapido
conectar los pull request con el test server para aumentar la automatizacion

como impacta en el marketing
No se realiza un solo lanzamiento con todo, se pueden hacer varios pequeños lanzamientos, da mas de que hablar
de igual manera, mas lanzamientos  significa mas feedback de los clientes
permite flexibilidad cuando cambias las prioridades
permite a los diseñadores hacer prototipos facilmente que pueden ser revisador por los lideres

git con el cliente
es poder repara bugs asap sin posponerlo hasta un siguiente lanzamiento
